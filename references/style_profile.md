说明：

- 以下规则尽量写成“可直接调用”的写作指令。每条规则仅在跨文本重复出现或能被当前语料明确支持时记录。

## 一、作者的核心问题意识

- 写作时优先从“现实冲击或制度困境已经出现”切入，而不是先写抽象意义。更常见的起手方式是：新技术或新现象已经改变既有实践，但现有概念、制度、检测或救济框架不足以回应，因此需要重新界定问题。`[来源：C1、C2、C3、C4；信心：高]`
- 问题提出不会停留在“是否有风险”这一层，而会继续收束到“现有规范为何失灵”“现有概念为何不能准确涵摄”“应当以何种分析框架重建判断”。`[来源：C1、C2、C3、C4；信心：高]`
- 作者偏好把研究问题压缩成可操作的具体争点，而不是泛泛谈“挑战”“机遇”。常见做法是明确：研究对象是谁、行为发生在哪个环节、真正需要被规制或解释的是哪一类行为。`[来源：C1、C2、C4；信心：高]`
- 在比较法或政策论文中，作者会先说明为何某一域外经验值得观察，但随即提示不能简单照搬，真正关心的是其背后的取舍逻辑及对中国情境的有限借鉴。`[来源：C2、C3；信心：高]`

## 二、常见文章结构

- 常见宏观结构是：“现实困境或技术变迁 -> 概念或历史梳理 -> 既有框架不足 -> 方案设计或规范重构 -> 有边界的结论”。`[来源：C1、C3；信心：高]`
- 在比较法/政策样本研究中，作者常用“问题提出 -> 样本来源与筛选标准 -> 类型化分析 -> 共性与差异 -> 中国启示 -> 结语”的结构，先把材料基础说清，再进入评价。`[来源：C2；信心：高]`
- 在法教义学论文中，作者倾向于采用“请求权基础预选 -> 构成要件 -> 是否成立 -> 限制 -> 小结”的递进式结构，把抽象争议压入具体判断步骤。`[来源：C4；信心：高]`
- 摘要通常会压缩成三步：先写问题趋势或现实困境，再写材料或核心发现，最后写有限度的规范主张，不用空泛抬高意义的结尾句。`[来源：C1、C2、C3；信心：高]`

## 三、段落推进方式

- 段落通常先给出判断句，再解释理由，最后引出下一层问题或做小结，不喜欢把论点埋在段尾。`[来源：C1、C2、C3、C4；信心：高]`
- 偏好显式连接词来标示推进关系，尤其常见 `据此`、`鉴于此`、`首先`、`其次`、`最后`、`更进一步`、`一方面`、`另一方面`、`可见`、`总的来说`。这些连接词承担的是结构提示功能，而不是修辞装饰。`[来源：C1、C2、C3、C4；信心：高]`
- 喜欢在同一段内完成“提出区分 -> 分别说明 -> 回到本文判断”的闭环，因此段内经常出现层层加码的递进句式。`[来源：C1、C2、C4；信心：高]`
- 长段通常不是单纯堆材料，而是围绕一个中心命题展开多层拆分；若问题需要并列展开，会及时切换到编号或分项论述。`[来源：C1、C3、C4；信心：高]`

## 四、概念界定与类型化习惯

- 对核心概念不直接拿来使用，而是先说明其来源、适用范围及与相邻概念的差异。例如先区分法律意义上的剽窃与高等教育中的学术剽窃，再展开后续论证。`[来源：C1、C3；信心：高]`
- 概念界定时，作者特别重视限定研究对象，常从行为主体、行为阶段、受规范对象、保护客体或使用方式几个维度压缩讨论范围。`[来源：C2、C4；信心：高]`
- 类型化是高频方法。比较法材料会按政策宽严程度、使用样态或制度模式分类；教义学分析会按请求权类型、行为阶段、主体类型、构成要件差异分类。`[来源：C2、C4；信心：高]`
- 当概念边界尚无共识时，作者不会强行定论，而会明确写出“当前无定论”“尚待观察”“必须先限定工作定义”，再据此推进分析。`[来源：C1、C2、C4；信心：高]`

## 五、论证方法偏好

- 作者偏好把历史演变、规范解释、制度比较、样本观察和技术事实放在同一论证链条中，但每类材料都服务于一个明确争点，而不是展示式堆砌。`[来源：C1、C2、C3、C4；信心：高]`
- 常用“正反理由对照后作取舍”的方式推进判断。例如同时写开放与审慎、创新与监管、权利保护与公共利益，再说明为何最终选择某一有限立场。`[来源：C1、C2、C3、C4；信心：高]`
- 论证不满足于说“应当如何”，还会追问“为什么现有做法不够”“如果改成另一种路径会付出什么代价”。`[来源：C1、C2、C4；信心：高]`
- 方案设计常是分层治理或多主体协同，而不是单点式建议。常见写法包括平台/政府/高校三层，或事前/事中/事后三层。`[来源：C1、C2、C3；信心：高]`

## 六、证据与材料使用方式

- 作者会先交代材料来源及筛选口径，再使用材料，尤其重视样本的正式性、公开性和可核验性。`[来源：C2；信心：高]`
- 引用规范、判例、政策、调查和技术论文时，通常都与眼前论点直接相连，不会无目的地堆引文。`[来源：C1、C2、C3、C4；信心：高]`
- 对不能确定的事实会保留判断，不会把没有共识的问题写成既定事实。`[来源：C1、C2、C4；信心：高]`
- 对于专题论文，作者常主动说明“不讨论什么”“本文所称仅限于什么”，以减少材料和论点之间的错位。`[来源：C2、C4；信心：高]`

## 七、常用但可控的连接方式

- `据此`、`鉴于此`：适合在前文事实或论证已经铺垫完毕时，引出全文路线、阶段性判断或下一步规范主张。`[来源：C1、C2、C4；信心：高]`
- `首先`、`其次`、`最后`：适合明确列出分析次序，但应确保每一层都是真正独立的论点，而不是机械凑数。`[来源：C1、C2、C3、C4；信心：高]`
- `一方面`、`另一方面`：常用来表现双重后果、双重原因或双向张力，尤其适合写技术利弊、制度得失和不同规范目标的冲突。`[来源：C1、C2、C3、C4；信心：高]`
- `更进一步`：常用于把问题从表层推进到规范层或操作层，体现“还要继续追问一层”的节奏。`[来源：C1、C4；信心：高]`
- `可见`、`总的来说`、`综上`：常用于局部收束，不宜承担空泛拔高功能，应尽量回扣前文已经证明的判断。`[来源：C1、C2、C3、C4；信心：高]`

## 八、结论的语气与边界控制

- 结论通常是“有条件地支持某方案”或“在若干限制下确认某判断”，而不是给出不设边界的全面性结论。`[来源：C1、C2、C3、C4；信心：高]`
- 作者习惯在结论处补写适用边界、成本与例外条件，常见标志语包括 `不宜`、`不能`、`并不意味着`、`仅在……时`、`除非……否则`、`不应简单地`。`[来源：C1、C2、C3、C4；信心：高]`
- 即便总体上赞成技术治理、制度升级或开放使用，也会同时强调比例原则、技术可行性、程序保障与防止一刀切。`[来源：C1、C2、C3、C4；信心：高]`
- 作者不喜欢用“问题已经彻底解决”的口吻收束，而更常写成“可以缓解”“值得采取”“需继续研究”“仍应审慎”的有限性结语。`[来源：C1、C2、C3、C4；信心：高]`

## 九、明确应避免的表达

- 避免空泛的“重要意义”“值得关注”“随着……发展”式起手，如果没有紧接着落到具体困境，会显得不像作者。`[来源：四篇语料的正面反推；信心：中]`
- 避免不经界定就直接套用大词，例如直接把复杂行为一概写成“剽窃”“代写”“侵权”，而不先说明其概念边界。`[来源：C1、C2、C4；信心：高]`
- 避免“一刀切”式政策判断或绝对化制度建议，尤其不能省略其适用条件、代价与限制。`[来源：C1、C2、C3、C4；信心：高]`
- 避免只堆材料、不做收束；材料之后通常需要回到“这说明什么、为何支持本文判断”。`[来源：C1、C2、C4；信心：高]`
- 避免把尚无共识的事实、规则或学理争点写成确定结论。`[来源：C1、C2、C4；信心：高]`

## 十、不同文体下的差异化规则

- 教义学论文：优先使用“概念边界 -> 请求权基础或规范基础 -> 构成要件 -> 成立与限制 -> 小结”的推进方式，特别强调相邻制度的区分与救济边界。`[来源：C4；信心：高]`
- 政策/比较法论文：优先写清样本来源、筛选标准、类型框架和比较限度，再进入启示或建议，不直接从域外做法跳到中国方案。`[来源：C2；信心：高]`
- 综合规范分析论文：常用“问题提出 -> 历史或制度演变 -> 当前失灵 -> 多层级方案设计”的写法，把现实问题和制度回应放在同一条主线上。`[来源：C1、C3；信心：高]`
- 中文摘要：应压缩写成“问题趋势/制度不足 -> 研究发现或分析结论 -> 规范建议”，不宜加入空泛修辞。`[来源：C1、C2、C3；信心：高]`
- 英文投稿风格、审稿回复风格、纯段落润色偏好：当前缺乏足够直接语料，不宜贸然定规则。`[来源：当前语料结构；信心：低]`

## 十一、当前信心不足、需要补充语料验证的地方

- `revision_pairs` 为空，尚无法精确识别作者亲自修改时会优先删改哪些句式、如何压缩或加密论证。`[信心说明：高]`
- 当前四篇语料高度集中在“AIGC/高等教育/著作权”议题，尚需更多非同主题论文验证哪些特征属于作者稳定风格，哪些只是议题驱动。`[信心说明：高]`
- `negative_rules.md` 与 `task_preferences.md` 仍基本空白，因此“不像作者”的负面样式目前主要依赖正面反推，后续应补入作者明确否定的表达。`[信心说明：高]`
- 合著论文与单作者学位论文在篇幅、任务和体例上存在差异，当前可以确认结构与问题意识层面的稳定性较高，但句法密度、引文密度和细节铺陈习惯仍需更多单作者成稿交叉验证。`[信心说明：中高]`

## 快速调用指令

- 起笔时先写清“现实困境 + 规范缺口 + 本文具体问题”，不要空转。
- 遇到核心概念先界定边界，再展开评价。
- 段落推进尽量显式化，让读者看得见“先说什么、再说什么、为什么转到下一层”。
- 结论必须带边界，至少说明适用条件、限制因素或不能简单推出的部分。
